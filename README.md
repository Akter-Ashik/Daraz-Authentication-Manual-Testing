# Software-Quality-Assurance
Manual test cases for Daraz's Signup, Login, and Forgot Password pages. Includes positive, negative, and edge scenarios to ensure secure and smooth user authentication. Covers SMS verification, password rules, OTP flow, and error handling for reliable account creation and recovery.






Daraz Authentication Manual Testing

This repository contains comprehensive manual test cases for the following Daraz features:

Signup Page – User registration with phone number, SMS verification, password rules, and promotional opt-in validation.

Login Page – Secure user login flow validation including valid/invalid credentials and account lock scenarios.

Forgot Password Page – Account recovery via phone or email, including OTP verification and reset link validation.

Contents





signup_test_cases.xlsx → Test cases for new account creation.

login_test_cases.xlsx → Test cases for user login validation.

forgot_password_test_cases.xlsx → Test cases for password reset flow.

Highlights



✅ Positive and negative test coverage

✅ Validation for UI, functional, and error handling

✅ Edge cases such as rate limiting, OTP validation, and session timeout

✅ Ready-to-use format for manual QA teams
